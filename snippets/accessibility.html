A few elements to keep in mind

Aria

role="alert" aria-live="assertive"
  • aria-live="assertive - polite - off" - the Screen Reader should read all changes to the element
    • Looking like it is best to seperate the info you need to be read (expand / collapse) into a seperate div. https://accessibility.huit.harvard.edu/provide-notification-dynamic-changes-content

Role

  • role="alert" - this is more aggressive version of aria-live
  • role="presentation" - tells screen reader it should be skipped as it is only for astitics.
